# Agent Instructions for Basar

## Project Identity

This repo is `Basar`, a 0G-first evidence-preserving memory layer for AI agents and research teams.

## Required Security Boundary

Read `SECURITY_BOUNDARY.md` before copying, generating, or publishing any content.

Never copy private prior-project materials into this repo. This includes trading alpha, strategy results, risk parameters, wallet keys, API keys, secrets, private datasets, proprietary research conclusions, or project-specific operational artifacts.

Only generic infrastructure patterns may be reused:

- Research save workflows.
- Knowledge card generation patterns.
- Evidence registry structure.
- Validation metadata patterns.
- Memory kernel structure.
- Provenance tracking.
- Markdown-based research artifact formats.
- Generic agent or research workflow organization.

## Working Style

- Keep local project coordination notes in `docs/basar-handoff/`; this directory is intentionally ignored by git.
- Keep public-facing language focused on Basar, Zero Cup, 0G, evidence preservation, provenance, local-first archives, and user-owned AI compute.
- Do not perform deployment, GitHub push, destructive git operations, or secret-bearing integrations without explicit maintainer approval.
- Prefer small, reviewable changes.
- When unsure whether material is private, treat it as private and do not copy it.
