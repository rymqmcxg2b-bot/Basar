# Data Governance

The repository must not include copyrighted books, paywalled articles, private user documents, scraped private data, secrets, or private archives.

Example data must be public domain, CC0, self-authored, or small synthetic examples. Users are responsible for the legality of materials they ingest locally.

Local archives remain local by default. 0G backup or publishing must be explicit. Sensitive files should never be uploaded or decentralized without deliberate user action. Export and deletion workflows should be preserved as first-class project requirements.
